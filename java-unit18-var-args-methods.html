ΔΩΡΕΑΝ ΜΑΘΗΜΑΤΑ JAVA

Στο σημερινό δωρεάν μάθημα java θα μιλήσουμε για το πως μπορούμε να τροποποιήσουμε τις μεθόδους μας κατά τέτοιο τρόπο ώστε να δέχονται μεταβλητό αριθμό μεταβλητών.

Μέχρι τώρα έχουμε δει ότι για να δημιουργήσουμε σωστά μια μέθοδο θα πρέπει να ορίσουμε εξ αρχής όλες τις παραμέτρους που θα δέχεται. Η Java όμως μας δίνει την δυνατότητα να μπορούμε να δημιουργήσουμε μεθόδους στις οποίες ο αριθμός των μεταβλητών τους δεν είναι σταθερός. Αυτός ο όρος ονομάζεται variable-length arguments ή αλλιώς var-args. Ο καλύτερος τρόπος όμως για να εξηγήσουμε πως ακριβώς δομείται μια τέτοια μέθοδος αλλά και πως λειτουργεί είναι μέσω ενός παραδείγματος.

Ας δημιουργήσουμε λοιπόν ένα καινούργιο project και ας το ονομάσουμε VarArgsMethods. Το πακέτο το οποίο θα δημιουργήσετε θα ονομάζεται com.mycompany. Μέσα στο πακέτο θα δημιουργήσουμε μια κλάση με το όνομα MathUtil η οποία θα περιέχει τον εξής κώδικα:

MathUtil.java


    package com.mycompany;

    public class MathUtil {
        public int max(int x, int y){
            int max = x;
            
            if(y > max){
                max = y;
            }
            return max;
        }
    }

Σε αυτή την κλάση έχουμε δημιουργήσει μια απλή μέθοδο με το όνομα max η οποία, αφού δεχτεί δύο int αριθμούς θα μας επιστρέψει πίσω τον μεγαλύτερο από τους δύο αριθμούς. Μέχρι τώρα ακολουθούμε την θεωρία όπως ακριβώς την γνωρίζουμε.

Δημιουργούμε μια δεύτερη κλάση με το όνομα MathUtilDemo η οποία περιέχει τον εξής κώδικα:

MathUtilDemo.java

    package com.mycompany;

    public class MathUtilDemo {
        public static void main(String[] args) {
            MathUtil max_num = new MathUtil();
            int result = max_num.max(5, 10);
            System.out.println("The maximum number is: "+ result);
                     
        }
    }

Output

    The maximum number is: 10

Ούτε και αυτή η κλάση περιέχει κάτι καινούργιο. Δημιουργούμε ένα αντικείμενο από την κλάση MathUtil και αφού καλέσουμε την μέθοδο max περνώντας δύο int αριθμούς, λαμβάνουμε πίσω το αποτέλεσμα ποιος από τους δύο αριθμούς είναι ο μεγαλύτερος.

Τώρα παρουσιάζεται η ανάγκη να καλύψουμε την δυνατότητα να βρίσκουμε τον μεγαλύτερο αριθμό από τρεις αριθμούς. Με βάση την θεωρία την οποία γνωρίζουμε, αυτό που θα κάνατε θα ήταν να χρησιμοποιήσετε την λογική του overloading. Δηλαδή να δημιουργήσετε μια ακόμη μέθοδο με το ίδιο όνομα που θα δέχεται τρεις μεταβλητές. Η κλάση MathUtil τώρα θα έχει τον εξής κώδικα:

MathUtil.java

    package com.mycompany;

    public class MathUtil {
        public int max(int x, int y){
            int max = x;
            
            if(y > max){
                max = y;
            }
            return max;
        }
        
         public int max(int x, int y, int z){
             int max = x;
             
             if(y>max){
                 max=y;
             }
             if(z>max){
                 max=z;
             }
             return max;
        }
    }

Ενώ η κλάση MathUtilDemo μπορεί τώρα να καλέσει την max μέθοδο με τις τρεις παραμέτρους.

MathUtilDemo.java

    package com.mycompany;

public class MathUtilDemo {
    public static void main(String[] args) {
        MathUtil max_num = new MathUtil();
        int result = max_num.max(5, 10, 15);
        System.out.println("The maximum number is: "+ result);
                 
    }
}

Output

    The maximum number is: 15

Οπότε με το ίδιο σκεπτικό και με την θεωρία του overloading που γνωρίζουμε μέχρι τώρα λογικά θα πρέπει να γεμίζουμε την κλάση MathUtil με καινούργιες μεθόδους που ικανοποιούν τις καινούργιες απαιτήσεις. Αυτό σημαίνει ότι θα πρέπει ο προγραμματιστής να ξοδέψει έξτρα χρόνο απλά για να προσθέτει καινούργιες μεθόδους. Η java είδε αυτό το επαναλαμβανόμενο πρόβλημα, και για αυτό μας έδωσε την δυνατότητα να ορίζουμε μεταβλητό αριθμό μεταβλητών σε μια μέθοδο. Αυτό σημαίνει ότι αντί να δημιουργήσουμε μια μέθοδο που να δέχεται ένα int αριθμό, μια άλλη μέθοδο που θα δέχεται δύο int αριθμούς κτλ να αντικατασταθούν με μια μόνο μέθοδο. Ας δούμε λοιπόν πως θα μπορούσε να γραφτεί η MathUtil κλάση χρησιμοποιώντας var-args μεταβλητή στην μέθοδο.

MathUtil.java


    package com.mycompany;

    public class MathUtil {
        public int max(int...num){
            int max = 0;
            
            for (int i=0; i<num.length;i++){
                if(num[i]>max){
                    max=num[i];
                }
            }
            return max;
        }

Όπως βλέπεται, το να ορίσουμε μια μέθοδο να χρησιμοποιεί var-args μεταβλητή δεν είναι τίποτα άλλο από το να γράψουμε τρεις τελείες αμέσως μετά το data type και το όνομα της παραμέτρου. Αυτός ο τρόπος μας επιτρέπει να καλέσουμε την μέθοδο με όσες παραμέτρους τύπου int θέλουμε συμπεριλαμβανομένου και του μηδέν – δηλαδή να μην περάσουμε καμία τιμή στην μέθοδο. Φυσικά στο δικό μας παράδειγμα αυτό δεν έχει πολύ αξία, όμως στην λογική του var-args αυτό επιτρέπεται.

Αν προσέξετε τώρα τον κώδικα μέσα στο for loop, θα σας φανεί γνώριμος ο τρόπος με τον οποίο διαβάζουμε τον αριθμό των παραμέτρων και των τιμών τους που έχουν περαστεί στην μέθοδο. Εσωτερικά η Java αναγνωρίζει την var-args num μεταβλητή σαν array οπότε εμείς μπορούμε να χρησιμοποιήσουμε την λογική του index (όπως κάναμε και στους array) για να διαβάσουμε τις τιμές που έχουν περαστεί στην μέθοδο. Με την λογική αυτή λοιπόν, αν είχαμε περάσει τρεις τιμές, αυτές θα βρισκόντουσαν στο num[0], num[1], και num[2] αντίστοιχα. Επειδή όμως δεν γνωρίζουμε εξ αρχής πόσες τιμές θα περαστούν στην μέθοδο, για αυτό χρησιμοποιούμε το for loop.

Ας τρέξουμε πάλι την MathUtilDemo για να δούμε πως λειτουργεί το πρόγραμμα μας αν περάσουμε διαφορετικό αριθμό παραμέτρων κάθε φορά.

MathUtilDemo.java

    package com.mycompany;

    public class MathUtilDemo {
        public static void main(String[] args) {
            MathUtil max_num = new MathUtil();
            int result1 = max_num.max(5, 10, 15);
            System.out.println("The maximum number is: "+ result1);
            int result2 = max_num.max(5, 10, 15, 20);
            System.out.println("The maximum number is: "+ result2);
            int result3 = max_num.max();
            System.out.println("The maximum number is: "+ result3);
                     
        }
    }

Output

    The maximum number is: 15
    The maximum number is: 20
    The maximum number is: 0

Η Java μας θέτει δύο περιορισμούς για το πως μπορούμε να ορίσουμε var-args μεθόδους.

1. Μπορούμε να χρησιμοποιήσουμε μόνο μια var-args μεταβλητή στην μέθοδο. Οπότε απαγορεύεται να κάνουμε κάτι όπως αυτό:

public int max(int...num1, int...num2){

//Code

}

2. Μπορούμε να χρησιμοποιήσουμε και άλλες παραμέτρους στον ορισμό της μεθόδου αρκεί όμως η var-args μεταβλητή να είναι τελευταία στην λίστα. Δηλαδή, μπορούμε να γράψουμε κάτι όπως:

public int max(int num1, int num2, int...num3){

//Code

}

Βασικά ο δεύτερος κανόνας εκτός από απαίτηση της java είναι και χρήσιμος γιατί με αυτό τον τρόπο μπορούμε να ζητήσουμε στα σίγουρα ένα συγκεκριμένο αριθμό δεδομένων (πχ. δύο τιμές) ενώ οποιοσδήποτε αριθμός τιμών πάνω από δύο να είναι προαιρετικός.

Τώρα ας θεωρήσουμε ότι καλύψαμε με μια var-args μέθοδο όλες τις int περιπτώσεις αλλά χρειαζόμαστε και τις double περιπτώσεις αριθμών. Οπότε χρειαζόμαστε ακόμα την λογική του overloading. Αυτό ισχύει με τις var-args μεθόδους? Φυσικά και ισχύει αρκεί να ικανοποιείται ο εξής απλός κανόνας – ότι το data type των παραμέτρων πρέπει να είναι διαφορετικός σε κάθε μια από τις μεθόδους.

MathUtil.java


    package com.mycompany;

    public class MathUtil {
        public int max(int...num){
            int max = 0;
            
            for (int i=0; i<num.length; i++){
                if(num[i]>max){
                    max=num[i];
                }
            }
            return max;
        }
        
        
        public double max(double...num){
            double max = 0;
            
            for (int i=0; i<num.length; i++){
                if(num[i]>max){
                    max=num[i];
                }
            }
            return max;
        }
    }

Υπάρχει όμως και η λογική ότι θα επιθυμούσατε να κάνετε overloading μια var-args μέθοδο με μια κανονική μέθοδο. Όπως το παρακάτω παράδειγμα:

MathUtil.java

    package com.mycompany;
    public class MathUtil {

        public int max(int x, int y){
       //Code
        }
   
       public int max(int...num){
             //Code
       }

Μπορείτε να μαντέψετε ποια από τις δύο μεθόδους θα ενεργοποιηθεί αν καλέσουμε την max(2, 4)? Η java θα κοιτάξει πρώτα να δει αν υπάρχει κάποια μέθοδο με τον ακριβή αριθμό παραμέτρων και data type που μπορεί να εκτελέσει τις τιμές που περάστηκαν στην μέθοδο. Αν υπάρχει (που στο παράδειγμα μας υπάρχει) τότε θα εκτελεστεί αυτή η μέθοδος. Αν δεν υπάρχει τότε η java θα ανατρέξει στην var-args μέθοδο.

Τέλος πριν κλείσουμε την ενότητα θα ήθελα να αναφερθώ στον τρόπο που πολλοί καινούργιοι προγραμματιστές γράφουν την μέθοδο main. Η main, όπως γνωρίζεται ήδη, δέχεται έναν array είδος String. Αυτός είναι ο πρώτος τρόπος με τον οποίο σχεδίασαν την java να μπορεί να δέχεται παραμέτρους ( τιμές με άλλα λόγια) από τον χρήστη κατά την διάρκεια εκτέλεσης του προγράμματος. Δηλαδή να μπορεί ο κώδικας σας να ρωτάει τον χρήστη να εισάγει κάποιες τιμές πριν συνεχίσει την εκτέλεση του. Τώρα που γνωρίζετε ότι εσωτερικά μια var-args μεταβλητή διαβάζεται από την java σαν array, μην σας φανεί παράξενο αν κάποιος γράψει την main κάπως έτσι.

MathUtilDemo.java

    package com.mycompany;

    public class MathUtilDemo {
        public static void main(String...args) {
            MathUtil max_num = new MathUtil();
             int result = max_num.max(1,2,3);
            System.out.println("The maximum number is: "+ result);
        }
    }
