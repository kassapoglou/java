ΔΩΡΕΑΝ ΜΑΘΗΜΑΤΑ JAVA

Στο σημερινό δωρεάν μάθημα Java θα δούμε πως μπορούμε να εισάγουμε τιμές στο πρόγραμμα μας ενώ αυτό τρέχει.

Τα παραδείγματα που έχουμε δει μέχρι τώρα περιέχουν τιμές που είναι ήδη δηλωμένες μέσα στον κώδικα. Τις περισσότερες φορές όμως, επιθυμούμε να έχουμε την δυνατότητα να περνάμε εμείς τις τιμές στο πρόγραμμα ενώ αυτό τρέχει. Αυτό συνήθως γίνεται με την εμφάνιση κάποιου μηνύματος στην οθόνη που μας ζητάει να εισάγουμε ένα είδος τιμής. Φυσικά, όλες οι μοντέρνες εφαρμογές παρουσιάζουν στην οθόνη του χρήστη κάποιο είδος φόρμας για να εισάγει δεδομένα. Στα δικά μας παραδείγματα θα χρησιμοποιήσουμε το cmd terminal των Windows.

Μέχρι τώρα έχουμε χρησιμοποιήσει την System.out.println( ) για να δείξουμε στον χρήστη κάποια μηνύματα της εφαρμογής. Καλώντας System.out στην ουσία ζητάμε να δείξουμε το αποτέλεσμα στο default output που είναι η κονσόλα των Windows. Υπάρχει και το αντίστοιχο System.in που μπορεί να δεχτεί δεδομένα από την default input συσκευή του υπολογιστή που είναι το πληκτρολόγιο. Το System.in όμως δεν είναι καθόλου εύχρηστο γιατί διαβάζει μόνο δεδομένα σε μορφή bytes. Εμείς όμως επιθυμούμε να μπορούμε να περάσουμε στο πρόγραμμα μας δεδομένα οποιασδήποτε μορφής όπως double, int, char κτλ.

Ευτυχώς οι μηχανικοί της Java είδαν αυτή την αδυναμία και δημιούργησαν μια καινούργια κλάση με το όνομα Scanner για να μας δώσουν έναν πιο ευέλικτο τρόπο εισαγωγής δεδομένων. Αφού η Scanner είναι κλάση, θα πρέπει να δημιουργήσουμε ένα αντικείμενο από αυτήν ώστε να μπορούμε να έχουμε πρόσβαση στις μεθόδους που μας προσφέρει. Κάθε κλάση όπως γνωρίζουμε προσφέρει τουλάχιστον έναν default no-argument constructor. Η Scanner κλάση προσφέρει αρκετούς διαφορετικούς constructors ανάλογα από που θέλετε να διαβάσει τα δεδομένα. Για παράδειγμα από το πληκτρολόγιο ή από κάποιο αρχείο. Για να δείτε τους constructors όπως και τις μεθόδους της κλάσης Scanner δεν έχετε παρά να πάτε στο επίσημο site της Oracle.

https://docs.oracle.com/javase/10/docs/api/java/util/Scanner.html

Εμείς επιλέγουμε να χρησιμοποιήσουμε τον constructor που δέχεται InputStream αντικείμενο σαν μεταβλητή. Ποιο αντικείμενο γνωρίζουμε που μπορούμε να περάσουμε άμεσα και έχει σαν default input το πληκτρολόγιο? Μα φυσικά το System.in. Οπότε για να δημιουργήσουμε ένα αντικείμενο από την Scanner χρησιμοποιούμε την εξής κώδικα:

Scanner input = New Scanner(System.in);

Η λίστα με τις μεθόδους που είναι διαθέσιμη για την μεταβλητή input είναι αρκετά μεγάλη. Μπορείτε άλλωστε να το δείτε και στο documentation αυτό.

Γιατί όμως υπάρχουν τόσες πολλές μέθοδοι? Γιατί η java ζητάει από εμάς κατά την εισαγωγή δεδομένων να γνωρίζουμε ήδη το data type των τιμών που εισάγουμε. Οπότε αποφεύγουμε έτσι τυχόν λάθη αλλά και άσκοπες μετατροπές μέσα στο κώδικα μας.

Και πως χρησιμοποιούμε τις μεθόδους? Βασικά, για αρχή πρέπει να γράψουμε ένα μήνυμα στον χρήστη για να γνωρίζει τι ακριβώς θα πρέπει να εισάγει στο πρόγραμμα. Αμέσως μετά καλούμε μια από τις μεθόδους του αντικειμένου in. Το αποτέλεσμα το αναθέτουμε σε μια καινούργια μεταβλητή ίδιου data type με εκείνο που μας επιστρέφει η μέθοδο. Για παράδειγμα για να ζητήσουμε το όνομα του χρήστη θα μπορούσαμε να γράψουμε:

System.out.print("What is your name? ");

String name = in.nextLine();

Η μέθοδο nextLine( ) δέχεται String σαν input. Οπότε αφού διαβάσει αυτό που θα εισάγει ο χρήστης, για να μπορέσουμε να το διαχειριστούμε προγραμματιστικά, το αναθέτουμε σε μια μεταβλητή είδος String.

Με την ίδια λογική θα μπορούσαμε να δεχτούμε και μια int τιμή. Αν κοιτάξετε το documentation θα δείτε ότι αυτό μπορεί να γίνει αν καλέσουμε την μέθοδο nextInt( ). Για παράδειγμα:

System.out.print("How old are you? ");

int age = in.nextInt();

Πριν δούμε ένα απλό παράδειγμα, πρέπει να αναφέρουμε ότι θα πρέπει να κάνουμε import την κλάση Scanner από το πακέτο java.util αφού η Java δεν την περιέχει μέσα στις κλάσεις που φορτώνει αυτόματα όταν δημιουργούμε ένα πρόγραμμα. Οι default κλάσεις που φορτώνει η Java περιέχονται μέσα στο πακέτο java.lang. Οποιαδήποτε κλάση που δεν υπάρχει σε αυτό το πακέτο πρέπει να την δηλώσουμε εμείς.

Employee.java

package scanner;

import java.util.Scanner;

public class TaxCalculator {

    public static void main(String[] args) {
        double RATE1 = 0.10;
        double RATE2 = 0.25;
        double RATE1_SINGLE_LIMIT = 32000;
        double RATE1_MARRIED_LIMIT = 64000;

        double tax1 = 0;
        double tax2 = 0;

        Scanner in = new Scanner(System.in);
        System.out.print("Please enter your income: ");
        double income = in.nextDouble();

        System.out.print("Please enter s for single, m for married: ");
        String maritalStatus = in.next();

        if (maritalStatus.equals("s")) {
            if (income <= RATE1_SINGLE_LIMIT) {
                tax1 = RATE1 * income;
            } else {
                tax1 = RATE1 * RATE1_SINGLE_LIMIT;
                tax2 = RATE2 * (income - RATE1_SINGLE_LIMIT);
            }
        } else {
            if (income <= RATE1_MARRIED_LIMIT) {
                tax1 = RATE1 * income;
            } else {
                tax1 = RATE1 * RATE1_MARRIED_LIMIT;
                tax2 = RATE2 * (income - RATE1_MARRIED_LIMIT);
            }
        }

        double totalTax = tax1 + tax2;

        System.out.println("The tax is $" + totalTax);
    }
}

Output

    Please enter your income: 35000
    Please enter s for single, m for married: m
    The tax is $3500.0

Στην αρχή του προγράμματος, ανάμεσα στον ορισμό του package και της class, θα δείτε ότι κάνουμε import την κλάση Scanner για να μπορεί να αναγνωριστεί σωστά ο ορισμός της όταν την χρησιμοποιήσουμε μέσα στον κώδικα μας.

Αφού ορίσουμε κάποιες μεταβλητές, ζητάμε από τον χρήστη να μας εισάγει έναν double αριθμό που αντιστοιχεί στο εισόδημα του και έναν χαρακτήρα που αντιπροσωπεύει την οικογενειακή του κατάσταση. Με βάση αυτές τις δύο πληροφορίες κάνουμε κάποιους απλούς υπολογισμούς για να υπολογίσουμε ποιος θα είναι ο φόρος για το συγκεκριμένο εισόδημα.

Αν ποτέ χρειαστεί να ζητήσετε από τον χρήστη να εισάγει κάποιο password τότε η Scanner δεν είναι και η ποιο κατάλληλη κλάση αφού δεν κρύβει τους χαρακτήρες που τυπώνει ο χρήστης. Για αυτό το συγκεκριμένο σκοπό υπάρχει μια άλλη κλάση που ονομάζεται Console και θα μπορούσατε να την χρησιμοποιήσετε κάπως έτσι:

Console cons = System.console();

String username = cons.readLine("User name: ");

char[] passwd = cons.readPassword("Password: ");

Για λόγους ασφαλείας, η readPassword( ) επιστρέφει έναν array από χαρακτήρες και όχι ένα String. θα ήταν λοιπόν συνετό όταν επεξεργαστείτε το password μέσα στον κώδικα σας, να γυρίσετε πίσω και να γεμίσετε τον array με τυχαίες τιμές.
