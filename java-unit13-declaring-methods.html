ΔΩΡΕΑΝ ΜΑΘΗΜΑΤΑ JAVA

Στο σημερινό δωρεάν μάθημα Java θα αναφερθούμε στις μεθόδους και πιο συγκεκριμένα θα δούμε πως συντάσσονται, τι είναι το signature των μεθόδων, πως τις καλούμε από ένα αντικείμενο, πως περνάμε δεδομένα με την μορφή παραμέτρων, και τέλος πως λαμβάνουμε πίσω αποτελέσματα.

Όπως ήδη έχουμε αναφέρει στην προηγούμενη ενότητα, οι μέθοδοι εκφράζουν την συμπεριφορά του αντικειμένου – δηλαδή το τι μπορούμε να κάνουμε με το αντικείμενο. Μπορούμε όμως να δημιουργήσουμε και μεθόδους που δεν ανήκουν σε κάποιο αντικείμενο και απλά τις καλούμε αυτόνομα. Αυτή είναι η static ιδιότητα και σε αυτή την κατηγορία ανήκει και η main( ) μέθοδο που ήδη χρησιμοποιείται στα προγράμματα σας. Η χρησιμότητα της μεθόδου είναι ότι δίνουμε όνομα στην ομαδοποίηση κάποιων γραμμών κώδικα που συνήθως έχουν ένα κοινό σκοπό. Αν θέλω να καλέσω πολλαπλές φορές αυτή την ομάδα γραμμών κώδικα απλά καλώ το όνομα της μεθόδου μέσα στην οποία ανήκουν.

Η γενική δομή της μεθόδου είναι η εξής:

[modifiers] () [throws-clause] {

// Body of the method goes here

}

Πριν όμως πούμε περισσότερα για την θεωρία, ας προσθέσουμε μερικές μεθόδους στον κώδικα της κλάσης Employee που είχαμε δημιουργήσει στην προηγούμενη ενότητα. Ο κώδικας μας είναι τώρα ο εξής:

Employee.java

    package com.mycompany;

    class Employee{
        public String name;
        public int AFM;
        public int employeeId;
        public double salary;
        public String department;
    
        public void mailCheck(){
            System.out.println("Mailing a check to " + name);
        }
    
        public double payment(){
            return salary/52.0;
        }
    
        public int vacationCalc(int daysTotal, int daysRequested){
        int remainingDays = daysTotal - daysRequested;
        System.out.println("You have requested " + daysRequested + " days of vacation");
        return remainingDays;
        }
    }

Ας ξεκινήσουμε την ανάλυση μας από την πρώτη μέθοδο με το όνομα mailCheck. Κάθε μέθοδο πρέπει να έχει ένα όνομα και να ακολουθείται από παρένθεση. Η ονομασία της μεθόδου, που στο δικό μας παράδειγμα είναι mailCheck( ), ακολουθεί τους ίδιους γενικούς κανόνες όπως εκείνων των μεταβλητών. Αμέσως πριν από το όνομα πρέπει να δηλώσουμε το data type της τιμής που θα μας επιστρέφει η μέθοδος. Η mailCheck( ) δεν επιστρέφει τίποτα πίσω στο πρόγραμμα απλά εκτυπώνει ένα μήνυμα στην οθόνη. Για να δηλώσουμε ότι δεν περιμένουμε από την μέθοδο να μας επιστρέψει κάποια τιμή πίσω στο πρόγραμμα χρησιμοποιούμε την λέξη κλειδί void. Για να μπορούμε να έχουμε πρόσβαση στην μέθοδο από οποιαδήποτε άλλη κλάση, δηλώνουμε την πρόσβαση σαν public. Τα άλλα είδη access specifiers (ή modifiers) είναι private, protected και default access. Ακριβώς τα ίδια όπως και στις μεταβλητές. Με τα access specifiers ή modifiers θα ασχοληθούμε εκτενέστερα σε μελλοντική ενότητα. Για τώρα χρησιμοποιούμε public και στις instance μεταβλητές και στις μεθόδους έτσι ώστε όταν δημιουργήσουμε το αντικείμενο μας να έχουμε πρόσβαση σε όλες τις μεταβλητές και μεθόδους του αντικειμένου. Τέλος, αν η μέθοδος περιέχει έναν πιο πολύπλοκο κώδικα (π.χ άνοιγμα και διάβασμα αρχείου) υπάρχει περίπτωση να μην εκτελεστεί σωστά και να μας επιστρέψει λάθος, αν για παράδειγμα, λείπει το αρχείο. Σε αυτή την περίπτωση θα πρέπει ο κώδικας μας να περιέχει και κώδικα διαχείρισης λαθών ή όπως αλλιώς ονομάζονται try-catch. Δεν χρειάζεται να μπούμε σε αυτή την κατηγορία της θεωρίας ακόμα. Αυτό όμως που πρέπει να προσθέσουμε είναι ότι ολόκληρος ο κώδικας οποιασδήποτε μεθόδου περιέχεται μέσα σε άγκιστρα ( { } ).

Η δεύτερη μέθοδο Payment( ) ακολουθεί την ίδια δομή με την πρώτη μέθοδο με την διαφορά ότι αντί για void θα μας επιστρέψει μια double τιμή πίσω στο πρόγραμμα. Επίσης επειδή η μέθοδος θα μας επιστρέψει τιμή είναι απαραίτητη η χρήση της λέξης κλειδί return. Εδώ κάνουμε return το αποτέλεσμα μιας αριθμητικής πράξης ή όπως αλλιώς ονομάζεται expression. Θα μπορούσαμε να είχαμε εκτελέσει την πράξη νωρίτερα μέσα στην μέθοδο και απλά να κάναμε return την μεταβλητή μόνο που θα είχε το τελικό αποτέλεσμα.

Η τρίτη και τελευταία μέθοδος vacationCalc( ) χρησιμοποιεί όλα όσα έχουμε αναφέρει μέχρι τώρα. Επιστρέφει έναν int αριθμό δια μέσω της μεταβλητής remainingDays όταν εκτελεστεί. Επιπλέον δέχεται και δύο μεταβλητές – την daysTotal και την daysRequested. Αυτές οι μεταβλητές είναι local μεταβλητές τις μεθόδου που σημαίνει ότι υπάρχουν αποκλειστικά και μόνο μέσα στην μέθοδο vacationCalc( ). Οι δύο αυτές παράμετροι ορίζονται η κάθε μια ξεχωριστά με το δικό της data type. Όταν καλέσουμε την μέθοδο θα πρέπει να δώσουμε τιμές σε αυτές τις δύο παραμέτρους ειδάλλως δεν θα εκτελεστεί το πρόγραμμα μας.

Σε κάθε μέθοδο η πρώτη γραμμή κώδικα που την δηλώνει και περιλαμβάνει το access modifier, το return type, το όνομα και της παραμέτρους ονομάζεται method signature. Οπότε για την μέθοδο payment( ) το signature της μεθόδου είναι:

public double payment()

Αυτή είναι μια πληροφορία που αργότερα θα μας φανεί πολύ χρήσιμη όταν μιλήσουμε για method overloading.

Αφού ολοκληρώσαμε την κλάση Employee θα πρέπει τώρα να καλέσουμε αυτές τις μεθόδους και να τις χρησιμοποιήσουμε. Ας δούμε πως αλλάζει τώρα ο κώδικας για την EmployeeDemo κλάση και ποιο είναι το αποτέλεσμα της εκτέλεσης του προγράμματος και μετά θα αναλύσουμε την θεωρία.

EmployeeDemo.java

    package com.mycompany;

    public class EmployeeDemo {
        public static void main(String []args){
            Employee michail = new Employee();
            michail.AFM = 12345;
            michail.department = "IT";
            michail.employeeId = 100;
            michail.name = "Michail Kassapoglou";
            michail.salary = 10000;
    
            michail.mailCheck();
            double payment = michail.payment();
            System.out.println("Your montly payment is "+ payment);
            int vacation = michail.vacationCalc(22, 5);
            System.out.println("You still have "+ vacation +" of vacation days left");
            
        }
    }

Output

    Mailing a check to Michail Kassapoglou
    Your montly payment is 192.30769230769232
    You have requested 5 days of vacation
    You still have 17 of vacation days left

Δημιουργήσαμε ένα instance (αντικείμενο) από την κλάση Employee και ορίσαμε ένα reference (michail) για να έχουμε πρόσβαση στις μεταβλητές και μεθόδους του αντικειμένου. Σας υπενθυμίζω ότι δύο είναι οι λόγοι που μας επιτρέπουν να βλέπουμε και τις μεθόδους και τις μεταβλητές: Πρώτον γιατί και οι δύο κλάσεις είναι στο ίδιο πακέτο, και ο access modifier είναι public. Αργότερα θα δούμε πως η αλλαγή του access specifier μπορεί να επηρεάσει την πρόσβαση μας στα χαρακτηριστικά και τις ιδιότητες του αντικειμένου.

Αφού λοιπόν δημιουργήσουμε ένα reference με το όνομα michail και γράψουμε το όνομα και τελεία ( δηλαδή michail.) θα εμφανιστεί μια λίστα με όλες τις διαθέσιμες μεταβλητές και μεθόδους. Επιλέγουμε εξ αρχής να εκτελέσουμε την mailCheck( ) γράφοντας michail.mailCheck( ). Η συγκεκριμένη μέθοδο δεν επιστρέφει τίποτα πίσω στο πρόγραμμα μας απλά εκτυπώνει ένα μήνυμα στην κονσόλα.

Αμέσως μετά καλούμε την payment( ) μέθοδο. Επειδή αυτή η μέθοδο επιστρέφει τιμή, ορίζουμε μια μεταβλητή payment στην οποία θα ανατεθεί η τιμή της payment( ) μεθόδου όταν εκτελεστεί. Έπειτα εκτυπώνουμε την τιμή της payment μεταβλητής στην κονσόλα.

Τέλος, έχουμε την vacationCalc( ) μέθοδο που για να εκτελεστεί σωστά περιμένει να τις περάσουμε int τιμές. Ορίζουμε την τιμή 22 για τις συνολικές ημέρες διακοπών που έχουμε και τον αριθμό 5 για τις ημέρες που θέλουμε να χρησιμοποιήσουμε. Το αποτέλεσμα της μεθόδου που επιστρέφει έναν int αριθμό το αναθέτουμε σε μια μεταβλητή vacation και αμέσως μετά εκτυπώνουμε την τιμή της στην κονσόλα.

Έχουμε πολλά ακόμα να πούμε για τις μεθόδους στις επόμενες ενότητες. Η κατανόηση τους θα μας βοηθήσουν σε μια ομαλότερη εισαγωγή στους constructors.
