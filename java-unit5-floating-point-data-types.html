ΕΝΟΤΗΤΑ 5 JAVA SE - Floating Point Data Types

Michail Kassapoglou
27
Nov

ΔΩΡΕΑΝ ΜΑΘΗΜΑΤΑ JAVA

Εκτός από τους ακέραιους αριθμούς, μπορούμε να ορίσουμε μια Java μεταβλητή να δέχεται και δεκαδικούς αριθμούς. Δύο από τα οκτώ primitive types χρησιμοποιούνται για δεκαδικούς αριθμούς: float και double.

Η μόνη και βασική διαφορά ανάμεσά τους είναι το εύρος των αριθμών που μπορούν να καλύψουν αφού ένας float αριθμός περιγράφεται από 32 bits ενώ ένας double από 64 bits.

Στην προηγούμενη ενότητα είχαμε περιγράψει ότι η  default συμπεριφορά της Java με τους ακέραιους αριθμούς ήταν να προσπαθήσει να τους μετατρέψει όλους σε int εκτός και αν βάλουμε το L στον τέλος του αριθμού χαρακτηρίζοντας τον σαν Long. Στους δεκαδικούς η default συμπεριφορά είναι το double data type.  Αν θέλουμε να ορίσουμε έναν αριθμό να είναι float,  εκτός από το float data type μπροστά από το όνομα της μεταβλητής, πρέπει να προσθέσουμε και ένα F στο τέλος του αριθμού.

Γενικότερα το double είναι εκείνο που χρησιμοποιούμε πιο πολύ αφού έτσι και αλλιώς όλες οι μαθηματικές βιβλιοθήκες της Java χρησιμοποιούν και επιστρέφουν  double αριθμούς.

Ας δούμε όμως ένα παράδειγμα στο οποίο μπορούμε να δούμε ξεκάθαρα την χρήση των δύο αυτών data types  και που πρέπει να προσέχουμε όταν τα χρησιμοποιούμε σε αριθμητικές πράξεις. Προσέξτε και έναν καινούργιο τρόπο που μας επιτρέπει η Java να ορίσουμε double αλλά και int αριθμούς με την χρήση του underscore ( _ )που μας επιτρέπει να ξεχωρίζουμε γρήγορα τις δεκάδες, τις εκατοντάδες, χιλιάδες κτλ αντί να χρησιμοποιήσουμε κόμμα ( , ). Το underscore δεν το λαμβάνει υπόψη η Java όταν το πρόγραμμα γίνεται compile. Απλά είναι μια γρήγορη βοήθεια στο να ορίζουμε μεγάλους αριθμούς. Η αλήθεια είναι ότι δεν έχω δει ποτέ κανέναν να χρησιμοποιεί αυτό τον τρόπο εγγραφής μεγάλων αριθμών.

Δημιουργούμε ένα καινούργιο project και το ονομάζουμε FloatDemo. Μέσα στο project, δημιουργούμε ένα πακέτο με το όνομα com.mycompany και μέσα στο πακέτο μια κλάση με το όνομα FloatDemo που περιέχει τον εξής κώδικα:

FloatDemo.java

package com.mycompany;

public class FloatDemo {
    public static void main(String []args){
        double pi = 3.14159;
        float f = 2.7F;
        System.out.println("pi = " + pi);
        System.out.println("f = " + f);
        int n = 15, d = 4;
        f = n / d;
        System.out.println("15/4 = " + f);
        int radius = 10;
        double area = pi * radius * radius;
        System.out.println("area = " + area);
        double num = 9_324_794_268.0;
        System.out.println("num = " + num);
    }
}

Output

pi = 3.14159
f = 2.7
15/4 = 3.0
area = 314.159
num = 9.324794268E9

Στο πιο πάνω πρόγραμμα, αν και ο Java κώδικας είναι σωστά γραμμένος, το αποτέλεσμα είναι λάθος. Ας το αναλύσουμε γραμμή προς γραμμή και θα δούμε πως μπορούμε να το διορθώσουμε.

Ορίζουμε εξ αρχής δύο μεταβλητές pi και f  με την pi να είναι double και την f να είναι  float. Παρατηρήστε το F στο τέλος του αριθμού 2.7 που είναι η τιμή που αναθέσαμε στην μεταβλητή f.

Αμέσως μετά τον ορισμό των μεταβλητών, απλά ζητάμε να δούμε τις τιμές που τους έχουμε αναθέσει στο default output.

Στην συνέχεια, ορίζουμε δύο int μεταβλητές την n και την d με τιμές 15 και 4 αντίστοιχα. Και στο επόμενο βήμα εκτελούμε την διαίρεση 15/4 και ζητάμε να δούμε το αποτέλεσμα στο terminal. Όμως παρατηρούμε ότι το αποτέλεσμα είναι λάθος. Τι έγινε? Η απάντηση είναι απλή: Η διαίρεση μεταξύ δύο int αριθμών μας δίνει αποτέλεσμα  int δηλαδή 3. Επειδή όμως η μεταβλητή f έχει οριστεί σαν float, η Java πριν αναθέσει την int τιμή στην μεταβλητή  f την μετατρέπει σε float. Οπότε το 3 μετατρέπεται σε 3.0. Για να λάβουμε το σωστό αποτέλεσμα πρέπει πολύ απλά να μετατρέψουμε τον ένα από τους δύο int σε float με την διαδικασία του casting. 

f = (float) n / d;

Τότε αυτόματα και ο δεύτερο int αριθμός θα μετατραπεί σε float και η διαίρεση θα μας δώσει το σωστό αποτέλεσμα.

Τέλος ορίζουμε μια μεταβλητή num να είναι double και της αναθέτουμε την τιμή 9_324_794_268.0 στην οποία χρησιμοποιούμε underscore ( _ ) αντί για κόμμα ( , ) για να ξεχωρίζουν καλύτερα οι ομάδες των αριθμών. Όταν ζητάμε να δούμε με τι ισούται η μεταβλητή num, θα πάρουμε τον αριθμό 9.324794268E9. Τα underscore έχουν εξαφανιστεί ενώ το τελικό αποτέλεσμα η Java μας το δίνει σε συνδυασμό με την δύναμη του 10. Το Ε9 αντιπροσωπεύει με το 109.

FloatDemo.java

package com.mycompany;

public class FloatDemo {
    public static void main(String []args){
        double pi = 3.14159;
        float f = 2.7F;
        System.out.println("pi = " + pi);
        System.out.println("f = " + f);
        int n = 15, d = 4;
        f = (float) n / d;
        System.out.println("15/4 = " + f);
        int radius = 10;
        double area = pi * radius * radius;
        System.out.println("area = " + area);
        double num = 9_324_794_268.0;
        System.out.println("num = " + num);
    }
}

Output

pi = 3.14159
f = 2.7
15/4 = 3.75
area = 314.159
num = 9.324794268E9

Τώρα το αποτέλεσμα της διαίρεσης είναι σωστό.
