ΕΝΟΤΗΤΑ 19 JAVA SE - CONSTRUCTORS

Στην σημερινό δωρεάν μάθημα Java θα μιλήσουμε για τους constructors και πως μπορούμε να δημιουργήσουμε τους δικούς μας constructors για την δημιουργία αντικειμένων που δέχονται τιμές για τις instance μεταβλητές τους κατά την δημιουργία τους. Έχουμε αρκετή θεωρία να αναπτύξουμε οπότε ας τα πάρουμε τα πράγματα από την αρχή.

Για να μπορέσουμε να αναπτύξουμε την θεωρία μας θα χρησιμοποιήσουμε ένα παράδειγμα που αναφέρεται σε τραπεζικούς λογαριασμούς. Θα ξεκινήσουμε δημιουργώντας ένα project με το όνομα BankAccount και ένα πακέτο μέσα στο project με το όνομα com.mybank. Μέσα στο πακέτο δημιουργούμε μια κλάση με το όνομα Account.

Account.java

											
    package com.mybank;

    public class Account {
        private String name;
        private double balance;

        public String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }

        public double getBalance() {
            return balance;
        }

        public void setBalance(double balance) {
            this.balance = balance;
        }
    }

Η θεωρία της κλάσης Account σας είναι ήδη γνωστή. Περιέχει δύο private μεταβλητές για να μην έχουμε απευθείας πρόσβαση επάνω στις συγκεκριμένες μεταβλητές αλλά δια μέσω getter και setter μεθόδους. Αφήνουμε τον κώδικα των getter και setter όπως είναι αλλά θα μπορούσαν να περιέχουν και κάποια if-else statements για καλύτερο έλεγχο. Εμείς θα το αφήσουμε όπως είναι στο δικό μας παράδειγμα.

Τώρα δημιουργούμε την κλάση MyAccount μέσα από την οποία θα δημιουργήσουμε αντικείμενα είδος Account.

MyAccount.java

											
    package com.mybank;

    public class MyAccount {
        public static void main(String[] args) {
            Account account1= new Account();
            Account account2 = new Account();

            account1.setName("Michail");
            account1.setBalance(2000);
            account2.setName("John");
            account2.setBalance(1000);

      System.out.println(account1.getName()+" balance: "+account1.getBalance()+" Euro");
      System.out.println(account2.getName()+" balance: "+account2.getBalance()+" Euro");
        }
    }

Στην κλάση MyAccount ακολουθήσαμε την θεωρία όπως την ξέρουμε μέχρι τώρα, δηλαδή δημιουργήσαμε δύο αντικείμενα και μετά καλέσαμε τις μεταβλητές τους για να τους αναθέσουμε κάποιες τιμές.

Όπως ήδη γνωρίζουμε, από την στιγμή που έχουμε δημιουργήσει μια δική μας κλάση (πχ. Account) η Java μας προσφέρει έναν μηχανισμό που ονομάζεται constructor για να μπορέσουμε να δημιουργήσουμε αντικείμενα από την συγκεκριμένη κλάση. Αν και αυτός ο μηχανισμός είναι απαραίτητος για την δημιουργία αντικειμένων, εμείς θα ζητούσαμε κάτι παραπάνω. Αντί να δημιουργούμε πρώτα τα αντικείμενα και μετά να τους θέτουμε τις τιμές τους, θα επιθυμούσαμε να μπορούμε να το κάνουμε αυτό την στιγμή της δημιουργίας τους – δηλαδή μέσα στον constructor να περνούσαμε παραμέτρους όπως και στις μεθόδους. Αυτό η Java μας αφήνει να το πραγματοποιήσουμε γράφοντας δικούς μας constructors.

Πριν γράψουμε όμως τον δικό μας constructor, θα πρέπει να θυμόμαστε δύο απλούς κανόνες που τους χαρακτηρίζουν και στην ουσία τους κάνουν να διαφέρουν από τις μεθόδους

  • Ο constructor πάντα έχει το ίδιο όνομα με την κλάση
  • Δεν υπάρχει return type στον constructor ούτε καν το void

Έχοντας αυτούς τους δύο απλούς κανόνες υπόψιν, ας δημιουργήσουμε τον δικό μας constructor στην κλάση Account και ας δούμε πως αυτό επηρεάζει την MyAccount κλάση.

Account.java

											
    package com.mybank;

    public class Account {
        private String name;
        private double balance;

        public Account(String name, double balance){
            this.name = name;
            if (balance>0.0)
              this.balance=balance;
            else
              System.out.println("Sorry, I can not create an account with negative balance");
        }

        public void deposit(double depositAmount){
            if(depositAmount>0.0)
                balance=balance+depositAmount;
        }

        public String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }

        public double getBalance() {
            return balance;
        }
    }

Ο constructor Account, που προσφέρει η κλάση, δέχεται τιμές για δύο μεταβλητές – την name και την balance. Αυτό σημαίνει, όταν πάμε να δημιουργήσουμε το αντικείμενο μας, αν δεν δώσουμε τις δύο αυτές τιμές στον constructor δεν θα προχωρήσει στην δημιουργία του αντικειμένου. Με άλλα λόγια, απαιτούμε από εκείνον που θα κάνει αντικείμενο από την κλάση Account να μας δώσει δύο τιμές για την name και balance αντίστοιχα.

Μέσα στον constructor μπορούμε να γράψουμε κώδικα ελέγχου για οτιδήποτε θέλουμε. Στο δικό μας παράδειγμα, δεν κάνουμε κανένα έλεγχο για το name και αυτό που δεχόμαστε το περνάμε σαν επίσημη τιμή του αντικειμένου. Στην περίπτωση του balance όμως πριν αναθέσουμε την τιμή στην instance μεταβλητή balance του αντικειμένου απαιτούμε η τιμή που μας δίνεται να είναι θετική. Επίσης έχουμε αφαιρέσει την setBalance( ) μέθοδο γιατί θέλουμε το balance να οριστεί την στιγμή που θα δημιουργηθεί το αντικείμενο δια μέσω του constructor. Τέλος, έχουμε προσθέσει και μια ακόμα μέθοδο με το όνομα deposit η οποία δέχεται μια double τιμή, που πρέπει να είναι θετική, και προσθέτει το ποσόν σε αυτό που υπάρχει ήδη στην balance μεταβλητή.

Ας δούμε τώρα πόσο έχει αλλάξει ο κώδικας στην MyAccount κλάση.

MyAccount.java

											
    package com.mybank;

    public class MyAccount {
        public static void main(String[] args) {
            Account account1= new Account("Michail", 2000);
            Account account2 = new Account("John", 1000);

            account2.deposit(150);

         System.out.println(account1.getName()+" balance: "+account1.getBalance()+" Euro");
         System.out.println(account2.getName()+" balance: "+account2.getBalance()+" Euro");
        }
    }

Υπάρχει μια μεγάλη διαφορά από την προηγούμενη έκδοση της κλάσης. Όταν πάμε να δημιουργήσουμε το αντικείμενο καλώντας τον constructor με την λέξη new υπάρχει μόνο ο καινούργιος constructor που δημιουργήσαμε σαν μόνη επιλογή. Ο απλός constructor (ή no-argument constructor όπως ονομάζεται δεν υπάρχει πια.

Η Java από την στιγμή που δημιουργήσαμε έναν δικό μας constructor αποσύρει τον δικό της και δεν τον έχουμε πια σαν επιλογή. Φυσικά μπορούμε να τον προσθέσουμε πίσω και θα το δούμε αυτό σε λίγο. Αυτό που μας αφορά τώρα είναι ότι η μόνη επιλογή που έχουμε είναι να δημιουργήσουμε ένα αντικείμενο με κάποιες αρχικές τιμές. Αυτές οι τιμές είναι απαραίτητη προϋπόθεση για να συνεχίσει ο constructor στην δημιουργία του αντικειμένου.

Αφού δημιουργήσουμε δύο αντικείμενα μπορούμε, αν επιθυμούμε, να καλέσουμε και την deposit( ) μέθοδο για να προσθέσουμε χρήματα στον ήδη υπάρχοντα λογαριασμό. Και βασικά αυτό κάνουμε για το account2. Αυτός άλλωστε είναι και ο λόγος που αφαιρέσαμε την setBalance( ) από την κλάση Account γιατί θέλαμε να μην μπορεί κάποιος να διαφοροποιήσει το balance του λογαριασμού εκτός και αν κάνει επίσημα deposit. Οι τελικές γραμμές κώδικα της MyAccount απλά εκτυπώνουν στην οθόνη τις πληροφορίες του κάθε account.

Μπορούμε να έχουμε περισσότερες από μια επιλογή για την δημιουργία αντικειμένων? Φυσικά και μπορούμε, αν δημιουργήσουμε πολλαπλούς constructor στην κλάση Account. Μα πως είναι δυνατόν να έχουμε πολλαπλούς constructor στην ίδια κλάση αφού όλοι οι constructor έχουν το ίδιο όνομα με την κλάση? Μα φυσικά με την λογική του overloading. Κάθε constructor θα έχει διαφορετικό αριθμό και data type μεταβλητών στο signature. Ας πάμε λοιπόν να προσθέσουμε δύο ακόμα constructors – έναν που δεν χρειάζεται καμία παράμετρο (no argument) και έναν που δέχεται μια μεταβλητή.

Account.java

											
    package com.mybank;

    public class Account {
        private String name;
        private double balance;

        public Account(String name, double balance){
            this.name = name;
            if (balance>0.0)
              this.balance=balance;
            else
              System.out.println("Sorry, I can not create an account with negative balance");
        }

         public Account(double balance){

            if (balance>0.0)
               this.balance=balance;
            else
              System.out.println("Sorry, I can not create an account with negative balance");
        }

          public Account(){

        }

        public void deposit(double depositAmount){
            if(depositAmount>0.0)
                balance=balance+depositAmount;
        }

        public String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }

        public double getBalance() {
            return balance;
        }
    }

Τώρα όταν πάμε στην MyAccount να δημιουργήσουμε ένα αντικείμενο Account θα δούμε τρεις επιλογές όπως δείχνει η πιο κάτω εικόνα

Ανάλογα με την επιλογή μας θα χρειαστεί ίσως να προσθέσουμε και άλλες πληροφορίες στο αντικείμενο με τις setter και getter μεθόδους. Για παράδειγμα αν καλέσουμε τον constructor με το όνομα Account(double balance) αυτό σημαίνει ότι θα πρέπει λογικά να καλέσουμε, μετά την δημιουργία του αντικειμένου, και την setName(String name ) για να αποκτήσει ο λογαριασμός και όνομα ιδιοκτήτη.
